Question Answers

Define internet backbone in your own words.

Large data routes between massive networks and routers, often hosted by large government, commercial, or academic facilities

What is the difference between clients and servers on the internet?

clients are mostly the computers operated by people visiting a website, while the server is the website transferring and managing data to and from the clients. other relations exist, but follow much of the same structure.

Draw or describe how servers, routers, switches, work stations, and other hardware components are part of the internet.

They connect or host connections across the internet to other machines.

Describe the Internet Protocol Suite (TCP/IP) model of networking, in general terms, outlining how this protocol provides the data transfer mechanism required to establish client/server relationships.

There are different layers in the internet, going from your pc to the website you want to look at. Your PC needs to send a request through these layers, and then the server sends one back

List at least three examples of internet services that rely on client/server relationships.

sending an email, downloading a file from a website, watching a youtube video

What is hypertext?

a software system that links topics on the screen to related information and graphics, like linking a source or an image

How is “the web” related to the internet?

The internet is the massive system of interlinked machines, while the web is collections of websites that are accessible via the internet

Explain how the Hypertext Transfer Protocol (HTTP) is used to facilitate client/server interaction.

HTTP is used by the client to send a request to the server, and the server responds also in HTTP. used to transfer data from client to server

What is HTTP and how is it different from HTTPS?

http send plaintext messages, while https faster and encrypted, making it more trustworthy

Describe and illustrate the development of the Web in general terms using the Web 1.0, Web 2.0, and Web 3.0 generational paradigms.

The earliest days of the internet can be seen as Web 1.0, where websites were something to look at and browse. With Web 2.0, users can add new content and have a new level of interactiveness available to them. Web 3.0 is still developing and only just coming into play, and could be similar to Mark Zuckerberg's idea of 'the metaverse'.

Compare and contrast the Web 1.0, Web 2.0, and Web 3.0 stages of development.

web 1.0 = "read only web", made of static pages that are uninteractable

web 2.0 = "participative social web", average joe can add to and interact with content, like many social medias

web 3.0 = "read, write, execute web", more tools to search and edit, ML & AI integrations, decentralization. Making every user an owner rather than a user.

Describe the role markup languages play in the Web.

They tell browsers how to arrange and display content when rendering the page

Compare and contrast markup and scripting languages

markup languages handle how the website looks, while scripting gives the website most of its functionality

Explain the relationship between Standard Generalized Markup Language (SGML), Extensible Markup Language (XML), Hypertext Markup Language (HTML), Extensible Hypertext Markup Language (XHTML), Dynamic Hypertext Markup Language (DHTML), and markdown (MD).

SGML defines a lot of the rules that markup languages are built around. HTML is what most websites, like this one, use to display content so it looks nice and can use images, hyperlinks, javascript and more. XML is an alternative to HTML that provides more customizability than HTML does, as well as less bandwidth use
